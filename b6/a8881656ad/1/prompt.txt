刚刚的readme修改需要深刻体现如下内容：

  自动产品工厂 的意思是：Multica 的最小闭环不该停在“派单给 agent”，而是 想法 -> 规格 -> 项目仓库 -> 根任务 -> fanout 子任务 -> 实现/验证 -> 交付。
  所以它更像一个产品生产系统，而不是一个 AI 版 Jira。

  本地优先+云协同 的意思是：真正的智能内核应放在本地 runtime。群体智能、统一记忆、agent 自演进、对外部 agent CLI 的快速复用，都应该尽量在本地 daemon / runtime 层完成。云
  端只做轻控制平面：登录、工作区、状态同步、任务队列、可视化、远程入口。云越薄，演进越快。

  先适配现有 CLI 的意思是：Multica 不该重造一个封闭 agent，而是做“编排层”。Claude Code、Codex、OpenCode 是执行引擎；Multica 负责给它们补上 统一任务模型、统一记忆、统一
  policy、统一 repo/worktree、统一 swarm 协作、统一观察面。你要赢的不是单 agent 能力，而是“把多个现成强 agent 组织成一个可持续生产系统”的能力。
