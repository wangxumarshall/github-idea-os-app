# GitHub IdeaOS Mobile

Minimal Flutter scaffold for the GitHub IdeaOS mobile client.

Current scaffold includes:

- ideas list screen
- new idea screen
- idea editor screen
- API client structure for the existing Multica backend

Expected next steps:

1. add authenticated session storage
2. wire GitHub OAuth connect flow through the backend
3. connect autosave from editor to `/api/ideas/{slug}`
4. polish navigation and offline states
