Agenticos 继续坚持“**薄控制面 + 厚数据面**”的 Kubernetes 式哲学，不自建 AgentOS，而是精准胶合 HKUDS 四大成熟仓库（nanobot、ClawTeam、OpenHarness、OpenSpace），同时将 **claudecode（Claude Code）、opencode（OpenCode）、codex、trae-cli 等现有 CLI 工具** 全部作为**可热插拔外部执行器**。  

本次完善基于**2026 年最新学术·产业·X 顶级观点**深度研究：  
- **学术**：MemSkill（arXiv:2602.02474）、《Self-Evolving AI Agents 综述》（2025）、《Memory in the Age of AI Agents》——强调**可进化记忆技能（evolvable memory skills）**、体验式记忆（experiential memory）、技能库防“丰盛诅咒”（curse of abundance）的检索/选择机制、反射闭环。

### 1. 系统核心架构：4+1 模式（职责正交，已强化学术/产业模式）
- **入口收单层（Entrance）** —— **nanobot（原仓库完整保留）**  
  职责：多渠道（IM/CLI/Webhook）、鉴权、任务结构化 + MCP 发现。  
  **强化**：入口直接输出带 `eval_metrics` 的 Agenticos Protocol，支持 X 专家推荐的“先定义成功指标”。

- **路由与调度层（Orchestration）** —— **ClawTeam（总大脑）**  
  职责：Fast-path / Swarm-path 决策 + 任务拆解。  
  **重大强化（吸收 MaryamMiradi 15 模式 + LangGraph/CrewAI 最佳实践）**：  
  - 内置 orchestration_templates/ 支持：Supervisor/Worker（默认）、Hierarchical、MapReduce、Parallel Fan-Out、Debate（对立论证）。  
  - 执行器路由策略（claudecode → 推理/创意；codex → 确定性修改；opencode → 多角色并行；trae-cli → 低成本兜底）现在可动态由 skill_selection.py + eval 分数调整。

- **执行与运行时层（Execution）** —— **OpenHarness（唯一 MCP Client）**  
  职责：agent loop、memory、permissions、auto-compaction、React TUI。  
  **强化**：  
  - 强制集成 **MCP 协议**（Anthropic 标准，已被 OpenAI 等采纳）作为所有工具调用唯一通道。  
  - 内置 ReAct + Reflection 闭环（学术标配），支持 self-critique。  
  - CLI 执行器适配器全部要求：streaming JSON 输出（便于 trace）、非交互模式（--yes）、沙盒 + TTL 防死锁（产业 CLI 生产共识）。

- **知识与进化层（Evolution）** —— **OpenSpace**  
  职责：Skill 抽取、修复、lineage 可视化。  
  **重大强化（直接对标 MemSkill + Self-Evolving 综述）**：  
  - 将 SKILL.md 升级为**可进化记忆技能（evolvable memory skills）**：span-level 而非 turn-level，支持 experiential memory（从失败 trace 提纯）。  
  - 增加技能检索/选择机制（防 curse of abundance）：AgentPRM 式优先级 + embedding 向量索引（可选插件）。  
  - 异步批处理 + HITL + Eval-Driven 闭环：仅高质量 trace 触发深度 Replay。

- **+1 控制面（Control Plane）** —— **agenticos 主仓库（极薄自研）**  
  仅负责：Agenticos Protocol、Router、State Proxy、Observability、三大策略（调度、技能选择、**新增 Eval-Driven 评估策略**）。
